# 漫画半自動制作ツール

ブラウザで開いて使う、Umbrella Parade漫画制作の入力ツールです。

## 開くファイル

[[漫画半自動制作ツール.html]]

## できること

- キャラクター、タイトル、テーマ、4コマ台本を入力する
- Obsidian用の1投稿カードを生成する
- 画像生成プロンプトを生成する
- Instagram、X、TikTok用の投稿文を生成する
- Markdownファイルとして保存する
- 表示中の出力をコピーする

## 保存先の考え方

生成した画像は、出力された保存先に合わせて保存します。

- ヴェル13世：`Umbrella Parade/漫画/01_ヴェル13世/画像/未投稿`
- カーラ・マンソン：`Umbrella Parade/漫画/02_カーラ・マンソン/画像/未投稿`
- べるぼ：`Umbrella Parade/漫画/03_べるぼ/画像/未投稿`

投稿予約したら `予約済み`、投稿後は `投稿済み` に移します。

## キャラ別プリセット

このツールは、Obsidianに保存してあるキャラクター資料、漫画テンプレート、画像フォルダーからプリセットを作っています。

- ヴェル13世：キャラ設定、4コマ制作マニュアル、運用メモ、ネタ06〜15、人物画像を読み込み
- カーラ・マンソン：コンセプト・テーマ、漫画制作メモ、カーラの情緒ログ台本、人物画像を読み込み

ツール上でキャラクターを選ぶと、読み込み済み設定と参考画像が切り替わります。
「プリセット適用」を押すと、テーマ、狙い、サブコピーにそのキャラの初期値が入ります。
参考画像はプレビュー表示され、パスをコピーして画像生成時の参照に使えます。

## キャラ別プリセット欄の表示

キャラ別プリセット欄は、ヘッダー部分をクリックすると小さく畳んだり広げたりできます。
画像セットやギャラリー登録を編集したいときだけ開くと、画面を広く使えます。

## プリセット更新

Obsidian側のキャラクター資料や画像を大きく更新したら、Codexに「漫画ツールのキャラプリセットを更新して」と頼めば、このツール内のプリセットを再生成します。

- [[GitHub連携・配布メモ]]

自分用の制作版と、べるぼ教育事業の特典として配布する公開版は分けて管理します。

- ネタ台本ストック欄の折りたたみ表示
- キャラ別プリセット欄の折りたたみ表示
- AIモデル選択と手入力モデルID
- AI生成ネタのツール内ストック
- Obsidian URIで生成ストックへ送る機能
- 漫画画像作成プロンプトボタン
- OpenAI画像生成APIで漫画画像そのものを生成
- 生成画像をツール内でプレビュー
- 生成画像をキープ保存、ダウンロード、画像つき予約リスト追加

- 予約リストと月間カレンダー表示

HTML単体ではObsidianファイルへ完全自動追記はできないため、ツール内保存と `Obsidianへ送る` ボタンで運用します。


## 画像を追加したとき

ギャラリーの画像一覧は、ツール内に保存されたプリセットを使っています。
Obsidianに新しいキャラクター画像や漫画画像を追加したら、Codexに「漫画ツールのギャラリー画像リストを更新して」と頼んでください。

## キャラ別画像パターン

衣装違い、表情違い、ライブ衣装、雨の日衣装などを作りたい場合は、画像欄をセットしたあと `画像パターン` に名前を付けて `パターン保存` を押します。

例：

- `通常衣装`
- `ライブ衣装`
- `雨の日`
- `デフォルメ`
- `カーラ_ローズ衣装`

保存したパターンはキャラクターごとに分かれます。
ヴェル13世で保存したパターンはヴェル13世用、カーラで保存したパターンはカーラ用として扱います。

使うときは `保存済みパターン` から選んで `パターン読込` を押します。
画像生成プロンプトとObsidianカードにも、その画像パターン名が入ります。

## ツール上で画像を読み込む

`画像フォルダー読込` または `画像ファイル追加` で、ブラウザ上から画像を一時的にギャラリーへ追加できます。

注意点：

- 読み込んだ画像は、ブラウザ内のIndexedDBに保存され、同じPC・同じブラウザなら次回起動後も有効です。
- 読み込んだ画像は、同じPC・同じブラウザなら次回起動後も表示されます。
- Obsidianのフォルダー実体としても管理したい画像は、キャラクター画像フォルダーに入れたあと、Codexに「漫画ツールのギャラリー画像リストを更新して」と頼んでください。
- ブラウザの安全制限により、HTML単体ではObsidian Folderを自動で常時スキャンできません。

## ネタ台本ストック欄の表示

ネタ台本ストック欄は、漫画画像生成欄の上にあります。
ヘッダー部分をクリックすると、小さく畳んだり広げたりできます。
ネタを選んでから漫画画像生成へ進む流れで使います。

## 漫画画像をツール内で生成する

`漫画画像を生成` ボタンを押すと、現在の画像プロンプトをOpenAI画像生成APIに送り、生成された漫画画像をツール内で確認できます。

基本の流れ：

1. 上部のAPIプロバイダーを `OpenAI` にして、APIキーを保存します。
2. キャラクター、タイトル、4コマ台本、参考画像セットを整えます。
3. `漫画画像を生成` を押します。
4. 画像が良ければ `キープ保存`、手元に画像ファイルとして残すなら `画像をダウンロード` を押します。
5. そのまま投稿準備に進む場合は `この画像で予約リスト追加` を押します。

`この画像で予約リスト追加` は、まず表示中の画像情報を `予約投稿前` に追加し、そのあとキープ保存を試します。
キープ保存に失敗した場合でも、予約投稿前への追加は残ります。

キープ保存した画像は、このブラウザ内のIndexedDBに保存されます。予約リストには画像IDとファイル名が紐づきます。
本当のSNS自動予約投稿にはInstagram、X、TikTok側の認証と投稿API連携が必要なので、現時点では「投稿予定リストを作る」段階です。

注意点：

- APIキーは自分用ローカルツールのためにブラウザ保存しています。配布版ではサーバー経由にする必要があります。
- OpenAI画像モデルの利用には、アカウントや組織の利用条件・認証が必要な場合があります。`Your organization must be verified` と出たら、OpenAI Platformの `Organization > General` で `Verify Organization` を完了し、最大15分ほど待ってから再実行します。
- HTML単体では生成画像をObsidianの画像フォルダーへ直接自動保存できないため、必要に応じて `画像をダウンロード` して未投稿フォルダーへ移します。

## SNS予約投稿の完了管理

SNS予約投稿欄は `予約投稿前` と `予約投稿完了` に分かれています。

- 予約を追加すると、まず `予約投稿前` に入ります。
- 投稿予約が終わったものは、選択して `予約投稿完了に移動` を押します。
- 間違えて完了にした場合は、`予約投稿前に戻す` で戻せます。
- 月間カレンダーは、`予約投稿前` と `予約投稿完了` の両方を表示します。
- カレンダー上では、長いキャラクター名は `ヴェル`、`カーラ` などの短縮名で表示します。
- `予約投稿完了` に移動したものは、カレンダー上で `ヴェル済`、`カーラ済` のように `済` が付きます。
- `予約一覧MDコピー` と `予約一覧をObsidianへ送る` には、予約投稿前と予約投稿完了の状態も入ります。
- 予約リストはブラウザ保存容量を圧迫しないよう、カード全文や長い画像プロンプトではなく、予約管理に必要な軽い情報だけ保存します。

## OpenAIサポートに送る情報

画像生成でエラーが出た場合、ツール下部のエラー文に `x-request-id` が表示されることがあります。
OpenAIサポートから求められた場合は、その `x-request-id` とVerification画面の現在表示をそのまま送ります。

## 過去ネタとの重複を避ける

`ネタ台本ストック` に保存した過去ネタをもとに、現在のタイトル、テーマ、4コマ台本と似ているものをチェックできます。

- `過去ネタと重複しない` にチェックを入れると、AIネタ生成プロンプトに過去ストックの要約が入り、似た流れを避ける指示が追加されます。
- `重複チェック` を押すと、現在のネタと近い過去ネタ候補が表示されます。
- 似ている候補が出た場合は、タイトル、悩みの状況、3コマ目の転換、4コマ目の小さな行動のどれかを変えます。
- 漫画にしたネタは `作成済みに移動` を押して、未作成ネタと分けて管理します。
- もう一度作り直したい場合は `未作成に戻す` で戻せます。

Obsidianへ送った過去ネタも大事ですが、ツール内の重複チェックは `ネタ台本ストック` の保存データを見ます。
作ったネタはできるだけ `今の台本をストック` しておくと、重複回避が強くなります。

### ツール前の過去投稿を入れる

ツールを作る前に投稿した漫画も、`ツール前の過去投稿を登録` から作成済みストックへ入れられます。
タイトルだけでも重複チェックの材料になります。

画像から文字を読む機能は、このHTML単体ではまだ自動OCRではありません。
過去投稿画像をChatGPTやGeminiに読み込ませて、画像内のセリフやタイトルを文字起こしし、その文字を `メモ・テーマ` に貼ると重複チェックに使えます。

## OpenAI API認証が通らないときの代替運用

OpenAIのOrganization Verificationが通るまでは、APIでの自動画像生成だけ止まります。
その場合でもツール全体は使えます。

代替手順：

1. ツールの `プロンプトコピー` を押します。
2. ChatGPTなどの画像生成チャットに貼って漫画画像を作ります。
3. 生成画像をPCに保存します。
4. ツールの `外部生成画像を取り込む` を押して画像を読み込みます。画像をコピーしてツール上で貼り付けたり、プレビュー枠へドラッグ&ドロップして取り込むこともできます。
5. `キープ保存` または `この画像で予約リスト追加` を押します。

この方法なら、OpenAI APIのOrganization Verification待ちの間も、台本作成・画像確認・キープ・予約管理まで進められます。

## API以外で漫画画像を作る現実的な方法

API認証がない場合は、ツール内で直接画像を作るのではなく、画像生成は外部サービスに任せます。
このツールは、プロンプト作成・画像取り込み・キープ・予約管理に集中させます。

おすすめの流れ：

1. `プロンプトコピー` を押します。
2. ChatGPT、Gemini、Canva、ローカル画像生成AIなど、使える画像生成サービスに貼ります。
3. 生成画像をコピー、保存、またはドラッグします。
4. ツールに貼り付け・ドラッグ&ドロップ・ファイル選択で取り込みます。
5. `キープ保存` または `この画像で予約リスト追加` を押します。

## ChatGPTで画像生成するときの参考画像優先チェック

`ChatGPT添付画像優先` にチェックを入れると、画像生成プロンプトの先頭に「添付したキャラクター画像を最優先で使う」指示が入ります。

API認証が通っていない間や、ChatGPTのチャット画面にキャラクター画像を手動添付して漫画を作る場合は、チェックを入れたまま使います。
OpenAI APIで直接生成する場合や、添付画像を使わない場合はチェックを外します。

## SNS投稿文の本文ON/OFF

`SNS本文を入れる` のチェックを外すと、投稿文タブはハッシュタグのみになります。
べるぼさん用の通常運用では、チェックOFFでハッシュタグだけ投稿する想定です。

文章つきの投稿文が欲しい場合は、`SNS本文を入れる` にチェックを入れます。
配布版では、文章が欲しい人だけONにして使えます。

## 吹き出し配置が崩れるとき

ChatGPTなど外部の画像生成では、キャラクターの見た目は近づいても、4コマテンプレートの吹き出し位置が弱くなることがあります。
このツールの画像生成プロンプトでは、次の配置を最重要ルールとして入れています。

- 1コマ目：キャラ左、吹き出し右
- 2コマ目：キャラ右、吹き出し左
- 3コマ目：キャラ中央、吹き出し右
- 4コマ目：キャラ後ろ姿または中央寄り、吹き出し左、ロゴ右

生成結果で吹き出しが全部同じ側に寄った場合は、同じプロンプトで「吹き出し配置をテンプレ通りに修正」と追記して再生成します。

## 漫画内に出していい文字

テーマや狙いは、作者用の制作メモとしてツール内に残します。
ただし画像生成プロンプトでは、テーマや狙いを漫画内の文字として描かないようにしています。

漫画画像に入れていい文字は、タイトル、サブコピー、4コマのセリフ、ラストコピー、Umbrella Paradeロゴだけです。
「テーマ」「Theme」「狙い」「Aim」「メモ」「コンセプト」などの説明ラベルや、その本文は漫画画像に入れません。

## 背景色の変化

画像生成プロンプトには、4コマごとの背景色と感情の変化も入れています。

- 1コマ目：青。静かな悲しみ、迷い、雨の孤独。
- 2コマ目：紫。動けない感じ、自己否定、重さ。
- 3コマ目：赤。感情の転換、小さな勇気、少し強い表情。
- 4コマ目：オレンジ。あたたかい希望、最初の行動、雨上がりの小さな光。

雨のネオン街という世界観は共通にしつつ、コマごとに支配色を変えて感情の流れを見せます。

## カーラ専用の背景色

カーラ・マンソンはヴェル13世と背景の色合いを変えます。

- 1コマ目：ローズピンク。甘さ、不安、強がり。
- 2コマ目：ディープパープル。執着、疑い、抜け出せなさ。
- 3コマ目：ワインレッド。感情の反転、嫉妬、情念。
- 4コマ目：ダークゴールドまたは淡いピンク。弱さ、可愛さ、笑いの余韻。

吹き出し配置は共通ルールを使います。
1コマ目は右、2コマ目は左、3コマ目は右、4コマ目は左です。

## 画像登録の削除

各画像欄には `登録削除` または `削除` ボタンがあります。
押すと、その欄の画像登録を解除します。

注意点：

- Obsidian内の元画像ファイルは削除しません。
- `画像フォルダー読込` や `画像ファイル追加` で一時追加した画像を選んでいる場合は、その一時登録も削除します。
- まとめて外したい場合は `全画像登録解除` を使います。
- 生成画像のプレビューだけ外したい場合は `表示中画像を解除`、キープ保存済み画像を消したい場合は `キープ画像を削除` を使います。

## 追加読み込み画像の保存

`画像フォルダー読込` や `画像ファイル追加` で読み込んだ画像は、ブラウザ内のIndexedDBに保存されます。
同じPC・同じブラウザでこのツールを開けば、次回起動後も追加画像と画像セットを読み込めます。

注意点：

- Obsidianの元フォルダーへ自動コピーするわけではなく、ブラウザ内に画像データを保存します。
- 別のブラウザや別のPCには自動では引き継がれません。
- ブラウザのサイトデータを削除すると、保存した追加画像も消えることがあります。
- 保存済み追加画像をまとめて消す場合は `追加画像クリア` を使います。

## 漫画画像生成から予約投稿前へ送る流れ

1. 「漫画画像生成」欄の「制作する未作成ネタ」でネタを選ぶ。
2. 選んだネタが台本入力とプロンプトに反映される。
3. 「プロンプトコピー」または「API生成（認証済み向け）」で漫画画像を作る。
4. ChatGPTなど外部で生成した画像は「外部生成画像を取り込む」から表示する。
5. 画像を確認して「この画像で予約リスト追加」を押す。
6. 画像はキープ保存され、SNS予約投稿前リストに入り、選択中のネタは「作成済み・投稿済みネタ」へ移動する。

この流れにすると、未作成ネタ、作成済みネタ、予約投稿前の画像が混ざりにくくなります。

## 予約前ストック確認と日時変更

「作成済み・投稿済みネタ」と「キープ済み画像」は、SNS予約投稿の直前にある「予約前ストック確認」にまとめています。

- 作成済みネタは、過去に漫画化したネタの確認、読込、未作成への戻し、削除に使います。
- キープ済み画像は、表示、削除、そのまま予約投稿前リストへの追加に使います。
- SNS予約投稿では、予約リストから投稿を選び、「選択予約の日時変更」で予約日と予約時間を後から更新できます。
- 日時変更欄は一時入力用です。変更を反映したあと、または予約投稿をセットしたあとに自動で空になります。日時変更を反映した時は、変更後の日時の翌日が次の予約作成欄に入ります。
- X予約投稿をセットした時、今すぐX投稿した時、手動で予約投稿完了にした時は、次の投稿準備用に予約日が自動で翌日に進みます。予約時間は前回と同じ時間を引き継ぎます。
- 予約投稿が完了したら、漫画画像生成欄に表示中の画像は自動でクリアされます。キープ保存済み画像や予約に紐づいた画像データは残ります。

## 予約投稿するボタン

SNS予約投稿では、予約投稿前リストの投稿を選んで「予約投稿する」を押すと、選択した投稿が予約投稿完了へ移動します。
間違えて完了にした場合は「予約投稿前に戻す」で戻せます。

## Xへの投稿

SNS予約投稿で「予約投稿前」の投稿を選び、「今すぐXに投稿」を押すと、キープ済み画像をXへアップロードして画像つきポストを即時作成します。
投稿に成功した場合だけ、選択した投稿は「予約投稿完了」へ移動します。

「X予約投稿をセット」を押すと、選択した予約の予定日・予約時間を使って、ローカルX投稿サーバーの予約キューに登録します。
予定時刻になると、ローカルX投稿サーバーがXへ投稿します。
この方式はX本体の予約投稿ではなく、ローカルPC上の投稿サーバーが時間まで待つ仕組みです。
そのため、Xアカウント本体の予約投稿一覧には表示されません。
確認はツール内の「X予約キュー確認」と、予約投稿完了欄の `X予約済み` 表示で行います。
予約時刻にPCが起動していて、`X投稿サーバー起動.bat` が動いている必要があります。
「X予約投稿をセット」に成功した時点で、選択した投稿は予約投稿前から予約投稿完了欄へ移動します。
予約時刻前は、まだ実際の投稿が終わっていないため、完了欄の中で `X予約済み` と表示されます。
投稿に成功したあと「X予約キュー確認」を押すと、表示が `予約投稿完了` に更新され、Xの投稿IDやURLも反映されます。

### X予約/投稿を削除する

予約投稿完了欄で対象を選んで `X投稿/予約を削除` を押すと、状態に応じて削除処理を行います。

- まだ投稿時刻前の `X予約済み` は、ローカルX投稿サーバーの予約キューから削除し、ツール内では予約投稿前に戻します。
- 削除後はX予約キュー欄を自動更新し、残って見えないようにします。
- すでにXへ投稿済みでPost IDが保存されているものは、X APIでX上の投稿を削除します。
- 投稿済みなのにPost IDが表示されていない場合は、先に `X予約キュー確認` を押して同期してください。
- 削除後もツール内の履歴は消さず、`X予約削除済み` または `X投稿削除済み` として残します。

事前にX Developer Portalで、投稿するXアカウント本人として認可したOAuth 2.0 User ContextのUser Access Tokenを用意し、ツール内の「X投稿設定」に保存してください。
ツールの「Xログインしてトークン取得」を使うと、Xログイン画面からUser Context tokenを取得してキャラクター別に保存できます。
必要なOAuth 2.0スコープは `tweet.read`、`tweet.write`、`users.read`、`media.write`、`offline.access` です。
App only Access Token / Bearer Tokenは、ユーザー本人としての投稿権限を持たないため、このツールの投稿・トークン確認には使えません。

ブラウザからの直接通信がX側やブラウザの制限で止まる場合は、エラー表示を確認して、トークン権限・APIプラン・ブラウザの通信制限を切り分けます。

### キャラクター別Xトークン

X User Access Tokenはキャラクターごとに保存されます。
キャラクターを切り替えると、保存済みのトークンも自動で切り替わります。

- ヴェル13世を選んで保存したトークンは、ヴェル13世の投稿に使います。
- カーラ・マンソンを選んで保存したトークンは、カーラ・マンソンの投稿に使います。
- べるぼを選んで保存したトークンは、べるぼの投稿に使います。

予約投稿前リストから投稿する場合も、選択した予約のキャラクターに合わせたトークンを使います。

## XログインでUser Context tokenを取得する

1. X Developer Portalの対象アプリで、OAuth 2.0 / User authentication settingsを有効にします。
2. Callback URLに、ツール画面の「OAuth Callback URL」を完全一致で登録します。標準は `http://127.0.0.1:8787/x-callback` です。
3. ツールの「X OAuth Client ID」にDeveloper PortalのClient IDを入力します。
4. Web App / Automated AppなどClient Secretが必要なアプリの場合だけ、「X OAuth Client Secret」にClient Secretを入力します。Native App / Single page Appの場合は空で大丈夫です。
5. 「OAuth設定保存」を押します。
6. 投稿したいキャラクターを選んで、「Xログインしてトークン取得」を押します。
7. Xログイン画面で、投稿したいXアカウントでログインして許可します。
8. ツールに戻ると、自動でaccess_tokenとrefresh_tokenが保存されます。
9. ログイン後にツールへ戻れない場合は、漫画半自動制作ツールを開き直して「OAuth取得結果確認」を押します。X投稿サーバーが起動したままなら、直近の取得結果を回収できます。

`offline.access` を含めて取得できた場合、期限切れ前に「Xトークン更新」でaccess_tokenを更新できます。

補足:

- 一度OAuthログインで取得したaccess_token / refresh_tokenは、ブラウザのlocalStorageにキャラクター別で保存されます。
- ツールを開き直しても、保存済みならOAuth状態は `取得済み` または `取得済み/更新可` と表示されます。
- `取得済み/更新可` の場合は、access_tokenの期限が切れても `Xトークン更新` や投稿時の自動更新でログインし直しを減らせます。
- ブラウザの閲覧データ削除、別ブラウザ利用、別ファイルURLからの起動では保存が分かれることがあります。

## ローカルX投稿サーバー

ブラウザから直接X APIへ投稿できない場合に備えて、ツールはローカル投稿サーバー経由でXへ投稿します。

使い方:

1. `X投稿サーバー起動.bat` を開く。
2. ツールの「X投稿設定」で「投稿サーバー接続確認」を押す。
3. 接続OKになったら、予約投稿前リストから投稿を選ぶ。
4. すぐ投稿する場合は「今すぐXに投稿」を押す。
5. 予約時刻に投稿したい場合は「X予約投稿をセット」を押す。

「X予約投稿をセット」に成功した場合は、予約投稿前から予約投稿完了欄へ移動します。
「今すぐXに投稿」は、投稿に成功した場合だけ予約投稿前から予約投稿完了へ移動します。
投稿サーバーの標準URLは `http://127.0.0.1:8787` です。

## X投稿できない時の確認

X投稿が成功した場合だけ、予約投稿前から予約投稿完了へ移動します。
移動しない場合は投稿に失敗しています。

確認順:

1. 「投稿サーバー接続確認」で接続OKになるか確認する。
2. 「Xトークン確認」で、選択中キャラクターのトークンが有効か確認する。
3. 「最後の投稿エラー確認」で、X APIから返ってきたエラーを確認する。
4. 画像サイズが大きい場合は、ツールが投稿前に自動でJPEG圧縮します。

サーバー側の詳細ログは `x-post-server.log` に残ります。


## WordPress / SWELLへの同時投稿

WordPress投稿設定では、SWELLで使う「まだ、息してる！」用の投稿ルールを固定しています。

- 本文は漫画画像のみです。ハッシュタグやSNS本文は入りません。
- タイトルは `【まだ、息してる！】漫画タイトル` になります。
- 抜粋は `4コマ漫画「まだ、息してる！」漫画タイトル` になります。
- スラッグは `4comic-漫画連載回数` になります。
- カテゴリーは `4コマ漫画`、スラッグは `4-panel-comic` です。
- アイキャッチ画像は固定URLを使います。
- og:imageは、SEO SIMPLE PACKの `ssp_meta_image` とアイキャッチ画像の両方で設定します。

使い方:

1. WordPress管理画面でApplication Passwordを作る。
2. WordPress投稿設定の「設定対象キャラクター」で保存したいキャラクターを選び、サイトURL、ユーザー名、Application Passwordを保存する。
3. ヴェル13世、カーラ・マンソン、べるぼなど、サイトを分けたいキャラクターごとに同じ手順で保存する。
4. 漫画連載回数を入力する。
5. 予約投稿前または予約投稿完了欄から投稿を選ぶ。
6. `選択予約をWordPress投稿/予約` を押す。

WordPress投稿設定はキャラクターごとに保存されます。予約投稿時は、選択した予約のキャラクターに対応するWordPressサイト設定を使います。
最後に選んだ設定対象キャラクターも保存されるため、ツールを開き直した時も同じキャラクター用のWordPress設定が表示されます。
予約投稿では、ツールの予約日・予約時間をWordPressのサイト時間 `date` とUTC時間 `date_gmt` の両方に送ります。WordPress投稿状態が `予約投稿` になっていることを確認してください。
予約投稿後に「選択予約の日時変更」で日時を直した場合、WordPress投稿IDが保存されていて投稿状態が `future` のものは、WordPress側の予約日時も同時に更新します。

`X予約投稿をセットした時にWordPressにも投稿/予約` にチェックを入れると、そのキャラクター用の設定に従って、X予約登録と同時にWordPressにも同じ日時で予約投稿を作成します。

注意:

SWELLの表示上書き設定は `swell_meta_show_thumb=show`、`swell_meta_show_related=show`、`swell_meta_show_author=hide` を送ります。REST APIが成功しても未登録メタが画面に反映されないことがあるため、XML-RPC経由でも既存カスタムフィールドを上書きします。
SEO SIMPLE PACKを使っている場合、投稿・固定ページのog:imageは `ssp_meta_image` とアイキャッチ画像で反映します。

## Xトークン入力時の注意

X User Access Token欄には、OAuth 2.0の `access_token` 文字列だけを入力します。

- `Bearer ` は付けなくて大丈夫です。貼ってもツール側で自動的に外します。
- API Key、Client ID、Client Secret、Refresh Token、説明文は入れません。
- App only Access Token / Bearer Tokenではなく、OAuth 2.0 User Contextの `access_token` を使います。
- 日本語や全角文字が混ざっている場合は保存前にエラーになります。
