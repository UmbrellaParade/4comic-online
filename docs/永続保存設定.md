# 永続保存設定

Railway の通常デプロイ先である `/app/data` は、再デプロイ時に消える可能性があります。
X 予約投稿を確実に残すには、次のどちらかを設定します。

## 推奨: Railway Volume

Railway の Service に Volume を追加し、Mount path を `/data` にします。
その状態で `/health` の `persistence.dataDirPersistent` が `true` になれば、予約キューは再デプロイ後も残ります。

## 予備: GitHub 暗号化バックアップ

Railway Variables に以下を設定すると、X 予約キューを GitHub に暗号化して保存します。
予約キューには X のトークンが含まれるため、必ず `PERSISTENCE_ENCRYPTION_KEY` を設定します。

```text
PERSISTENCE_GITHUB_TOKEN=GitHub fine-grained token
PERSISTENCE_GITHUB_REPOSITORY=UmbrellaParade/4comic-online
PERSISTENCE_GITHUB_BRANCH=main
PERSISTENCE_SCHEDULE_PATH=.persistent/x-scheduled-posts.enc.json
PERSISTENCE_ENCRYPTION_KEY=長いランダム文字列
```

`/health` の `persistence.durable` が `true` なら、永続保存の準備ができています。

## 注意

- GitHub バックアップは暗号化済みの予約キューだけを保存します。
- `PERSISTENCE_ENCRYPTION_KEY` を失うと、GitHub に保存した予約キューを復元できません。
- WordPress の Application Password や X のトークンを平文で GitHub に置かないでください。
