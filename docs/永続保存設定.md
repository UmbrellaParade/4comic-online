# 永続保存設定

Railway の通常デプロイ先である `/app/data` は、再デプロイやサービス再作成で消える可能性があります。
予約投稿を安定させるには、次のどちらかではなく、用途に応じて両方を整えるのが安全です。

## 1. Railway Volume

Railway の Service に Volume を追加し、Mount path を `/data` にします。
Railway Variables では次を設定します。

```text
DATA_DIR=/data
VAULT_ROOT=/data/vault
```

この状態で `/health` の `persistence.dataDirPersistent` が `true` になれば、Railway 側の保存先は永続化されています。

## 2. GitHub Actions 予約実行

PCを起動していない状態でも X 予約投稿を動かすには、GitHub Actions が予約キューを読んで投稿する必要があります。
このリポジトリには `.github/workflows/x-scheduler.yml` があり、5分ごとに予約キューを確認します。

予約キューには X のトークンが含まれるため、GitHub に置くファイルは暗号化します。
GitHub の Repository Secrets に次を追加してください。

```text
PERSISTENCE_ENCRYPTION_KEY=長いランダム文字列
```

Railway Variables には次を設定します。

```text
PERSISTENCE_GITHUB_TOKEN=GitHub fine-grained token
PERSISTENCE_GITHUB_REPOSITORY=UmbrellaParade/4comic-online
PERSISTENCE_GITHUB_BRANCH=main
PERSISTENCE_SCHEDULE_PATH=.persistent/x-scheduled-posts.enc.json
PERSISTENCE_ENCRYPTION_KEY=GitHub Secrets と同じ文字列
```

`PERSISTENCE_GITHUB_TOKEN` は、対象リポジトリの Contents を read/write できる fine-grained token にします。

## 確認方法

`/health` を開いて、以下を確認します。

```text
persistence.durable = true
persistence.remoteSchedule.configured = true
```

GitHub の Actions タブで `X scheduled post runner` が成功していれば、オンライン側で予約実行できます。

## 注意

- コードを追加しただけでは永続化は有効になりません。Railway Variables、GitHub Secrets、Railway Volume の設定が別途必要です。
- `PERSISTENCE_ENCRYPTION_KEY` を失うと、GitHub に保存した暗号化予約キューを復元できません。
- WordPress Application Password や X のトークンを平文で GitHub に置かないでください。
