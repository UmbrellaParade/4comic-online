# オンライン化 設計メモ

## 方針

旧ツールの機能は残しつつ、投稿予約の実行場所をPCからオンラインサーバーへ移します。

## 初期版でコピーした機能

- キャラクター選択
- キャラ別プリセット
- ネタ自動生成
- ネタ台本ストック
- 漫画画像生成プロンプト
- OpenAI API画像生成
- 外部生成画像取り込み
- キープ済み画像
- SNS予約投稿前 / 予約投稿完了
- X OAuthログイン
- X投稿 / X予約投稿
- WordPress投稿 / 予約投稿
- SWELL/OGPメタ設定
- 予約カレンダー
- Obsidian向けMarkdown出力

## オンライン版で変えるところ

### サーバー

- `HOST=0.0.0.0` で外部アクセス可能にする。
- `PORT` はホスティングサービスの指定に従う。
- `PUBLIC_BASE_URL` をOAuth callbackの基準URLにする。
- `DATA_DIR` を永続ディスクに置く。

### 保存

初期版はファイル保存です。

- `data/x-scheduled-posts.json`
- `data/x-oauth-tokens.json`
- `data/x-post-server.log`

本格運用ではDB/Storageへ移行します。

- Supabase DB + Storage
- Cloudflare D1 + R2
- Render Disk + JSONのまま小規模運用

### スマホ対応

まずは既存HTMLを使い、あとから以下を直します。

- 左目次の折りたたみ
- ボタンのタップサイズ
- 予約投稿欄の縦並び
- 画像プレビューの縦長表示
- 長い設定欄の開閉

## 運用で必要なもの

- 公開URL
- X OAuth Client ID / Client Secret
- X Developer PortalのCallback URL
- WordPress Application Password
- OpenAI API Key
- 永続ディスクまたはDB

## 注意

旧ツールの秘密情報はこのフォルダーにコピーしません。オンライン環境では必ず環境変数またはオンラインサーバー上の安全な保存先で管理します。
